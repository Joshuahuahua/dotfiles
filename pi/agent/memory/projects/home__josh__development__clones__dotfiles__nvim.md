# Project memory
- Dotfiles repo uses shared base configs at repo root with OS-specific entrypoint wrappers. Mint-specific files live under `mint/`: `mint/.zshrc` dynamically resolves its own path and sources the shared root `.zshrc`; `mint/.tmux.conf` dynamically resolves `~/.tmux.conf` to find the repo root, then sources shared root `.tmux.conf`; `mint/alacritty.toml` imports `../alacritty/alacritty.toml` and sets Mint shell config. Shared configs remain at repo root (`.zshrc`, `.tmux.conf`, `alacritty/alacritty.toml`, `nvim/`). Neovim is currently shared directly via the root `nvim/` dir, not wrapped per-OS. The repo also now includes a managed Pi subset under `pi/agent/` for syncable Pi config/tooling (`AGENTS.md`, `settings.json`, extensions, and memory files), while `auth.json` and `sessions/` remain local-only and are not intended to be symlinked from the repo. `mint/install.sh` is the Mint bootstrap script and is intended to symlink Mint entrypoint files for zsh/tmux/alacritty plus shared `nvim/`, `lazygit.yml`, and the managed `pi/agent/` subset, but no Mint symlinks have been created yet in this session. Root `.zshrc` was also made more portable by replacing hardcoded `/home/josh/...` paths with `$HOME/...` for fnm, pnpm, and local bin paths. _(saved 2026-07-14)_
